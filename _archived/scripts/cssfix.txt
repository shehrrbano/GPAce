- Identify all CSS sources: external files, inline styles, and embedded `<style>` tags.

Step 2: **Analyze & Detect Redundancy**
- Detect exact and near-duplicate CSS rules.

Step 3: **Dependency & Conflict Resolution**
- Evaluate selector specificity and inheritance hierarchy.
- Detect potential conflicts that could cause visual regressions if merged.
- Apply rule consolidation only when it is safe (e.g., combining identical property-value pairs across selectors).

Step 4: **Optimization & Merging**
- Merge redundant CSS rules and group similar selectors.
- Convert inline styles to classes wherever safe.
- Ensure the merged CSS maintains the original styling and priority order.
